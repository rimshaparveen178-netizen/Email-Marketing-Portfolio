Portfolio · MMXXVI Methodology — Phase i.
Phase i. a. Foundation

Access first.
Architecture second.

Before a single email gets drafted, every new client begins with the same two-step setup — gather platform credentials and historical data, then organize every artifact into a sequenced folder system that's legible to humans and AI alike.

Step one is access. Klaviyo (admin), Google Analytics 4, Google Tag Manager, the relevant ad accounts, the eCommerce platform (Shopify or Magento), and where applicable BigQuery — all credentialed before anything else. Step two is the historical pull — three months of campaign data, flow performance, attribution, list health metrics. The audit needs context to be useful.

Only once those two things are in hand does the workspace get built. At the agency level, every brand has its own folder under OpiMax Projects > Brand. At the client level, that folder is numbered into a fixed sequence that mirrors the working order — the same sequence works for every client; only the contents change.

Fig. 13 OpiMax Projects > Brand — eighteen client workspaces under a single naming convention.
Fig. 14 Master Spa Parts internal structure — numbered folders mirror the working sequence: 1. Client Information · 2. Discounts & Promotions · 3. Performance Reports · 4. Previous Active Flows · 5. Previous Campaigns · 6. Forms & Popups · 7. Lists & Segments · onward through Deliverables.

A single Reference Document for AI sits at the root of each client folder, mapping what's in each subfolder. That reference is the entry point for the next phase — it's how the AI gets oriented to the brand without me having to re-explain every time.

Portfolio · MMXXVI Methodology — Phase ii.
Phase ii. b. AI-Augmented Audit

Audits, accelerated by
tested prompts.

The audit is where most of the strategic decisions get made — and where AI cuts the most time. Every client gets its own Claude Project with the full document base loaded as context, so the model isn't auditing in the dark.

Each Claude Project is set up the same way: the Reference Document for AI loads first as orientation; the brand strategy playbook, performance reports, and audit perspective files load next as source material; system instructions establish the analyst role and constraints. By the time the first audit prompt runs, the model already knows the brand, the data, and the deliverable format.

Prompt discipline matters. I keep a tested library of audit prompts — flow audit, segment audit, deliverability audit, copy audit, design audit — refined across multiple clients. New clients get the proven prompt; the model gets reproducible quality.

Fig. 15 Claude Projects index — one project per client, each preloaded with its document base and system instructions.
Fig. 16 Master Spa Parts project — partial view of audit threads spanning segments, deliverability, flow optimization, brand positioning, and campaign analysis.

Every audit ends in the same artifact: a gap analysis. What's broken, what's missing, what's underperforming, what's leaving money on the table. That document is what the entire next phase — strategy and planning — gets built on top of.

Phase iii. c. Strategy & Planning

From gaps to a plan.

Audit findings translate into two distinct planning tracks: lifecycle flow strategy — the always-on engine — and a monthly campaign calendar — the seasonal layer that runs on top. Lists & segmentation runs as a third track in parallel.

The flow track is structural. Each new flow gets a dedicated strategy document — trigger logic, branch conditions, message count, copy direction, exit rules, and the specific metric it's expected to move. Eleven flow strategies sit in the New Flows Strategy folder for Master Spa Parts alone — At-Risk Recovery, Chemical Replenishment, CLV Tiered, Cross-Sell, First-to-Second Purchase, Review Request, Sunset, Transactional, VIP Champion Retention, Welcome Series V2, plus the SMS Integration master.

The campaign track is rhythmic. Every month gets its own folder under Content Deliverables > Campaigns Plan, with the calendar built around seasonal moments, product launches, and segment-specific sends. Campaigns aren't drafted ad hoc — they're planned a month ahead, written two weeks ahead, and scheduled into Klaviyo two weeks before send.

Portfolio · MMXXVI Methodology — Phase iv.
Phase iv. d. Content Creation

Copy before design.

Every email starts as a structured Google Doc — subject lines (A and B), preview text, full body copy, and design notes for the team. The doc is the single source of truth before any pixel gets drawn.

The structure repeats across every email and every campaign series. For the Double Rewards campaign — a 4-day promotional series with three send touchpoints — the deliverable is one document with three sections: ANNOUNCEMENT, REMINDER, FINAL CALL. Each section has its own subject lines, preview text, full body copy with merge tags, and a designer brief at the bottom describing visual hierarchy, image direction, and CTA emphasis.

This separation matters. The copywriter's job is to make the message convert; the designer's job is to make it readable and on-brand. Mixing the two in a Klaviyo template editor before either is finalized is how programs get slow and inconsistent.

Fig. 19 _MSP_DOUBLE_REWARDS strategy doc — subject lines, preview text, body copy, and designer notes for all three emails in a single source file.

The copy doc is where the campaign is actually decided. By the time the designer opens Figma, every word, every CTA, every dynamic block has already been spec'd.

Portfolio · MMXXVI Methodology — Phase vi.
Phase vi. f. Implementation

Where strategy becomes
a live send.

Implementation is the final translation: every flow strategy doc becomes a flow chart in Klaviyo's visual builder; every email Google Doc becomes a tested template; every segment definition becomes a queryable list. This is where the time gets spent.

Email templates get sliced from the Figma design — image blocks, text blocks, dynamic product blocks (pulling from the custom feeds), CTA buttons all wired up. Variable blocks accept the merge tags from the copy doc directly. Every template gets a pre-send QA: render check, link check, dynamic-block check, deliverability check.

Fig. 22 Klaviyo email editor — the summer-ready educational campaign in production. Image blocks, text, products, CTAs all assembled directly from the brief.

Flows get architected in the visual builder — every trigger, every conditional split (if profile property X then path A else path B), every wait timer, every A/B variation node. The MSP CLV Tiered Flow uses 8+ branch conditions to send different messages to first-time, second-time, and high-value buyers, with separate cover, filter, and chemical paths inside each tier.

Fig. 23 Flow visual builder — branching logic for a multi-path tiered flow. Every condition, wait timer, and A/B variation visible at a glance.

Custom profile properties, segments, and lists — every audience definition is set up to be reusable across both flows and campaigns. A profile property created for the Replenishment flow becomes immediately available as a campaign filter for any send.
